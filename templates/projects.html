●Purpose — To gain practical experience implementing object-oriented solutions using optimal data structures.
● Discussion — This task scheduler uses priority structures, tuples, and pandas data frames to pull the data and provide an optimal, multi-taskable scheduler. Points to Note:
●This uses a priority queue because all tasks will be built inside the heap in an ordered manner; therefore, we only need to return the current tasks and the next tasks which are the root and the left child of the priority queue.
●This use immutable tuples in order to ensure that the relevant task IDs and priorities are not tampered with as the priority queue schedules the multitasking functionality.
●Every multitasked task calls heapify on the priority queue; therefore, big O(logn operation) will be called multiple times. Leave this code section option.
●How to use: Populate the CSV file with tasks to be executed or write them as lists with the name “tasks” (sample input will be attached below). Execute the Y loop which will give you a theoretical schedule for a full day based on the inputs provided. To use it as your own task scheduler, replace the step “variable” with a time.time counter which receives input from your world time and increment after every minute. Next, delete all print statements within the Y loop except for the task complete, task rescheduled, and mutli-taskable tasks to avoid cluttered output.
●Current Implementation — Fully complete.
